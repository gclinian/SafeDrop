SafeDrop

Zero-config file & clipboard sharing on your LAN — with a built-in cross-device MCP fabric so the AI agents on your laptop, phone, and Pi can see each other's tools.

View on GitHub Download MCP guide

What it does, in one breath

Run SafeDrop on two devices on the same Wi-Fi — they find each other, pair with a 4-digit code, and from then on you can drop files and clipboard between them end-to-end encrypted. Layer 2 is the cross-device MCP fabric: any AI agent (Claude Code, Cursor, Goose, Cline, Claude Desktop) on one device can call tools exposed by the others — read the phone's clipboard, take a photo with the phone's camera, run a shell command on a remote box — all over the same encrypted channel.

🔒End-to-end encrypted

X25519 ECDH → HKDF-SHA256 → Fernet (AES-128-CBC + HMAC-SHA256). Pair-code confirms the handshake. Same wire format byte-for-byte across Python / Kotlin / Swift.

🌐Zero-config discovery

UDP broadcast on the LAN. No router setup, no accounts, no QR codes. Manual-peer mode lets you connect across LANs with Tailscale or our discovery beacon.

🤖MCP fabric

Stdio for Claude Code / Cursor / Goose / Cline. HTTP Streamable-MCP with bearer tokens for cloud agents. Bridge other MCP servers under one namespace.

📱5 native clients

Python desktop GUI (tkinter), Kotlin Jetpack Compose Android app, Swift SwiftUI iOS app, headless CLI, MCP server. All speak the same protocol.

🧠Multi-agent mesh

Persistent agent_id per device. send_message / recv_messages / list_agents let agents on different boxes talk to each other autonomously.

🪞Trust + audit

Per-(peer, tool) decisions persisted to ~/.safedrop/trust.json. Every cross-device call lands in an append-only audit log. Capability tokens for scoped HTTP access.

30-second install

Desktop / CLI / MCP server install in one line. Android & iOS clients ship as APK / IPA on GitHub Releases. 

$ pip install -e 'safedrop[mcp]'           # desktop, CLI, MCP, tokens

# desktop GUI (needs tkinter)
$ python run.py

# headless CLI
$ safedrop ls
$ safedrop send-text macbook "hello from the terminal"

# expose as MCP tools to Claude Code
$ claude mcp add safedrop -- $(which safedrop-mcp)

# headless LLM bot reachable on the LAN (v1.4)
$ ANTHROPIC_API_KEY=... safedrop-agent --name-suffix agent

# cross-LAN discovery without a relay (v1.7)
$ safedrop tailscale list
$ safedrop-beacon --bind 0.0.0.0:47900 --secret $(openssl rand -hex 16)

The security model, in three lines

No magic — just well-understood primitives composed carefully.

1. Discovery is unauthenticated.

UDP HELLOs on the LAN carry the device's X25519 pubkey and TCP port. Anyone on the same LAN sees them. That's fine — pairing happens at TCP-time.

2. Each TCP session ECDH's a fresh Fernet key.

HKDF-SHA256 derives the symmetric key from the X25519 shared secret. The 4-digit pair code is derived from the same secret so both sides can verify the link out-of-band.

3. Cross-device tool calls go through a per-(peer, tool) trust gate.

Allow / Deny dialog on first use; "Always allow" persists to ~/.safedrop/trust.json. HTTP transport adds bearer-token auth with fnmatch-glob scopes.
